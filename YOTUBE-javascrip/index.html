Tipos de operadores

Que operadores tenemos?

**Operadores aritméticos**: Se utilizan para realizar operaciones matemáticas básicas.

* `+`: Suma dos valores.
* `-`: Resta el segundo valor del primero.
* `*`: Multiplica dos valores.
* `/`: Divide el primer valor por el segundo.
* `%`: Devuelve el resto de una división (operador módulo).

EJEMPLO:

      let l = 5 + (5 - 5) * 3;
      let division = 5 / 2;
      let modulo = 5 % 2;

      console.log(l);<=Devuelve 5
      console.log(division);<=Devuelve 2.5
      console.log(modulo);<=Devuelve 1
Nota:Al igual que en las matematicas los parentesis tiene mayor jerarquía.

Operadores de comparación: Se utilizan para comparar dos valores y devolver un valor booleano (true o false).


  • ==: Compara si dos valores son iguales.
  • !=: Compara si dos valores son diferentes.
  • ===: Compara si dos valores son iguales en valor y tipo (estrictamente iguales).
  • !==: Compara si dos valores son diferentes en valor o tipo (estrictamente diferentes).
  • >: Compara si el primer valor es mayor que el segundo.
  • <: Compara si el primer valor es menor que el segundo.
  • >=: Compara si el primer valor es mayor o igual que el segundo.
  • <=: Compara si el primer valor es menor o igual que el segundo.

Valores unarios


Operador de incremento (`++`):

  • El operador de incremento (`++`) aumenta el valor de una variable en uno.
  • Se puede colocar antes (`++variable`) o después (`variable++`) del nombre de la variable.
  • Si se coloca antes (`++variable`), primero se incrementa el valor de la variable y luego se utiliza en la expresión.
  • Si se coloca después (`variable++`), primero se utiliza el valor actual de la variable en la expresión y luego se incrementa.

EJEMPLO:

let contador = 0;
let resultado1 = ++contador; // Incrementa contador a 1 y luego asigna 1 a resultado1
let resultado2 = contador++; // Asigna el valor actual de contador (1) a resultado2 y luego incrementa contador a 2

Operador de decremento (`--`):

  • El operador de decremento (`--`) disminuye el valor de una variable en uno.
  • Al igual que con el operador de incremento, se puede colocar antes (`--variable`) o después (`variable--`) del nombre de la variable.
  • El comportamiento con respecto a la posición del operador es el mismo que con el operador de incremento.

EJEMPLO:

let contador = 0;
let resultado1 = --contador; // Decrementa contador a -1 y luego asigna -1 a resultado1
let resultado2 = contador--; // Asigna el valor actual de contador (-1) a resultado2 y luego decrementa contador a -2

Operador `+=` (suma y asignación):


  • El operador `+=` se utiliza para incrementar el valor de una variable sumándole otro valor y luego asignando el resultado a la misma variable.
  • La sintaxis es `variable += valor`, donde `variable` es la variable a la que se le asignará el resultado de la suma, y `valor` es el valor que se sumará a la variable.
  • Es equivalente a escribir `variable = variable + valor`.

EJEMPLO:

let numero = 5;

numero += 3; // Equivalente a: numero = numero + 3

// Ahora numero es 8

Operador `-=` (resta y asignación):


  • El operador `-=` se utiliza para decrementar el valor de una variable restando otro valor y luego asignando el resultado a la misma variable.
  • La sintaxis es `variable -= valor`, donde `variable` es la variable a la que se le asignará el resultado de la resta, y `valor` es el valor que se restará de la variable.
  • Es equivalente a escribir `variable = variable - valor`.

EJEMPLO:

let numero = 10;

numero -= 4; // Equivalente a: numero = numero - 4

// Ahora numero es 6

Operadores lógicos: Se utilizan para realizar operaciones lógicas entre valores booleanos.

  • `&&`: Operador AND. Devuelve `true` si ambos operandos son `true`.
  • `||`: Operador OR. Devuelve `true` si al menos uno de los operandos es `true`.
  • `!`: Operador NOT. Devuelve el valor opuesto del operando booleano.

Switch

Es una estructura de control que permite evaluar una expresión y ejecutar diferentes bloques de código según el valor de esa expresión. Es útil cuando tienes múltiples condiciones basadas en un solo valor.

EJEMPLO

    switch(EXPRESION) {
      case VALOR1:
        // Código si la expresión coincide con VALOR1
        break;
      case VALOR2:
        // Código si la expresión coincide con VALOR2
        break;
      default:
        // Código si no coincide con ningún caso
    }
  
    let dia = "Lunes";

    switch (dia) {
      case "Lunes":
        console.log("Hoy es lunes, inicio de semana.");
        break;
      case "Viernes":
        console.log("Hoy es viernes, casi fin de semana.");
        break;
      case "Sábado":
      case "Domingo":
        console.log("Es fin de semana, ¡a descansar!");
        break;
      default:
        console.log("Es un día normal de la semana.");
    }
    
      <= Si "dia" es "Lunes", me devolverá "Hoy es lunes, inicio de semana."
      Si cambio el valor a "Viernes", el mensaje será "Hoy es viernes, casi fin de semana."
      Para "Sábado" o "Domingo", el resultado será "Es fin de semana, ¡a descansar!"
      Si no coincide con ninguno, mostrará "Es un día normal de la semana."

Ciclos

While

El ciclo while repite un bloque de código mientras una condición sea verdadera.

EJEMPLO

      let i = 0;

      while (i < 5) {
        console.log("El número es: " + i);
        i++;
      }
      
        <= Este ciclo imprimirá los números del 0 al 4 mientras i sea menor que 5.

Do-While

El ciclo do-while ejecuta el bloque de código al menos una vez, incluso si la condición es falsa.

EJEMPLO

      let i = 0;

      do {
        console.log("El número es: " + i);
        i++;
      } while (i < 3);
      
        <= Este ciclo imprimirá los números del 0 al 2. La condición se evalúa al final.

For

El ciclo for se usa cuando conoces de antemano cuántas veces deseas repetir un bloque de código.

EJEMPLO

      for (let i = 0; i < 3; i++) {
        console.log("El número es: " + i);
      }
      
        <= Este ciclo imprimirá los números del 0 al 2.

For...in

El ciclo for...in itera sobre las propiedades enumerables de un objeto (o índices de un arreglo).

EJEMPLO

      let persona = { nombre: "Patrick", edad: 25 };

      for (let propiedad in persona) {
        console.log(propiedad + ": " + persona[propiedad]);
      }
      
        <= Este ciclo imprimirá "nombre: Patrick" y "edad: 25".

For...of

El ciclo for...of se utiliza para recorrer elementos de objetos iterables como arreglos, cadenas, etc.

EJEMPLO

      let numeros = [1, 2, 3];

      for (let numero of numeros) {
        console.log(numero);
      }
      
        <= Este ciclo imprimirá los números: 1, 2 y 3.

ForEach

El método forEach ejecuta una función para cada elemento de un arreglo.

EJEMPLO

      let numeros = [10, 20, 30];

      numeros.forEach((numero) => {
        console.log("Número: " + numero);
      });
      
        <= Este ciclo imprimirá "Número: 10", "Número: 20" y "Número: 30".

Try-Catch-Finally

La estructura try-catch-finally se utiliza para manejar errores en tiempo de ejecución. Permite intentar ejecutar un bloque de código y capturar errores si ocurren.

EJEMPLO

      try {
        let resultado = 10 / 0; // División por cero
        console.log("El resultado es: " + resultado);
      } catch (error) {
        console.log("Ocurrió un error: " + error.message);
      } finally {
        console.log("Bloque finally: Esto siempre se ejecuta.");
      }
      
        <= Este ejemplo imprimirá el resultado de la división (Infinity), no se generará un error real, pero el bloque finally siempre se ejecuta.

Creación de Errores Personalizados con throw

Con throw puedes lanzar tus propios errores personalizados. Es útil cuando deseas manejar situaciones específicas en tu código.

EJEMPLO

      function dividir(a, b) {
        if (b === 0) {
          throw new Error("No se puede dividir por cero.");
        }
        return a / b;
      }

      try {
        let resultado = dividir(10, 0); // Esto lanzará un error
        console.log("El resultado es: " + resultado);
      } catch (error) {
        console.log("Error: " + error.message);
      } finally {
        console.log("Bloque finally: Fin del programa.");
      }
      
        <= Este ejemplo lanzará un error con el mensaje "No se puede dividir por cero" y el bloque finally se ejecutará al final.

Break y Continue

Break

La instrucción break se utiliza para salir inmediatamente de un bucle o una estructura de control (como un switch), interrumpiendo su ejecución antes de que finalice normalmente.

EJEMPLO

      for (let i = 1; i <= 10; i++) {
        if (i === 5) {
          break; // Sale del bucle cuando i es igual a 5
        }
        console.log("Número: " + i);
      }
      
        <= Este ejemplo imprimirá los números del 1 al 4. Cuando i sea igual a 5, el bucle se interrumpirá y no continuará.

Continue

La instrucción continue se utiliza para omitir la iteración actual de un bucle y pasar a la siguiente, sin interrumpir el bucle completamente.

EJEMPLO

      for (let i = 1; i <= 10; i++) {
        if (i % 2 === 0) {
          continue; // Omite los números pares
        }
        console.log("Número impar: " + i);
      }
      
        <= Este ejemplo imprimirá solo los números impares del 1 al 10, ya que los números pares serán omitidos por el continue.

Desestructuración

¿Qué es la Desestructuración?

La desestructuración es una característica de JavaScript que permite extraer valores de arreglos o propiedades de objetos en variables individuales de manera sencilla.

Desestructuración de Arreglos

Con la desestructuración de arreglos, puedes asignar elementos de un arreglo a variables en una sola línea.

EJEMPLO

      const numeros = [1, 2, 3, 4];

      const [a, b, c] = numeros; // Asigna 1, 2 y 3 a las variables a, b y c

      console.log(a); // 1
      console.log(b); // 2
      console.log(c); // 3
      
        <= Este ejemplo asigna los tres primeros valores del arreglo numeros a las variables a, b y c.

Desestructuración de Objetos

Con la desestructuración de objetos, puedes asignar propiedades de un objeto a variables con el mismo nombre que las propiedades.

EJEMPLO

      const persona = {
        nombre: "Juan",
        edad: 30,
        pais: "México"
      };

      const { nombre, edad } = persona;

      console.log(nombre); // Juan
      console.log(edad); // 30
      
        <= Este ejemplo extrae las propiedades nombre y edad del objeto persona y las asigna a variables con el mismo nombre.

Desestructuración con Valores Predeterminados

Puedes asignar valores predeterminados en caso de que la propiedad o el elemento no exista.

EJEMPLO

      const persona = {
        nombre: "Juan"
      };

      const { nombre, edad = 25 } = persona;

      console.log(nombre); // Juan
      console.log(edad); // 25 (valor predeterminado)
      
        <= Este ejemplo asigna el valor predeterminado 25 a edad porque no existe en el objeto persona.

Objetos Literales

¿Qué son los Objetos Literales?

Los objetos literales son una forma sencilla de crear objetos en JavaScript. Se definen utilizando llaves {} y se componen de pares clave-valor.

Definición de un Objeto Literal

Un objeto literal es una estructura que contiene propiedades (clave-valor) y métodos (funciones dentro del objeto).

EJEMPLO

      const persona = {
        nombre: "María",
        edad: 28,
        pais: "España",
        saludar: function() {
          console.log("Hola, soy " + this.nombre);
        }
      };

      console.log(persona.nombre); // María
      console.log(persona.edad); // 28
      persona.saludar(); // Hola, soy María
      
        <= Este ejemplo define un objeto literal persona con tres propiedades (nombre, edad, pais) y un método (saludar).

Creación Dinámica de Propiedades

En los objetos literales, puedes asignar propiedades dinámicamente utilizando variables o expresiones.

EJEMPLO

      const propiedad = "edad";
      const persona = {
        nombre: "Carlos",
        [propiedad]: 35
      };

      console.log(persona.nombre); // Carlos
      console.log(persona.edad); // 35
      
        <= En este ejemplo, se utiliza una variable (propiedad) para asignar dinámicamente una propiedad al objeto persona.

Sintaxis Mejorada para Métodos

En los objetos literales, puedes definir métodos utilizando una sintaxis simplificada.

EJEMPLO

      const calculadora = {
        sumar(a, b) {
          return a + b;
        },
        restar(a, b) {
          return a - b;
        }
      };

      console.log(calculadora.sumar(5, 3)); // 8
      console.log(calculadora.restar(5, 3)); // 2
      
        <= Este ejemplo muestra cómo definir métodos en un objeto literal usando una sintaxis más concisa.

Parámetros REST y Operador Spread

¿Qué son los Parámetros REST?

Los parámetros REST permiten a las funciones aceptar un número indefinido de argumentos como un array. Se definen utilizando tres puntos consecutivos (...) antes del nombre del parámetro.

EJEMPLO

      function sumar(...numeros) {
        return numeros.reduce((acumulador, actual) => acumulador + actual, 0);
      }

      console.log(sumar(1, 2, 3, 4, 5)); // 15
      console.log(sumar(10, 20)); // 30
      
        <= Este ejemplo utiliza parámetros REST (...numeros) para recibir múltiples argumentos en forma de array y sumar sus valores.

¿Qué es el Operador Spread?

El operador spread (...) se utiliza para expandir elementos de un array, objeto u otro iterable en lugares donde se esperan argumentos o elementos individuales.

EJEMPLO

      const numeros = [1, 2, 3];
      console.log(Math.max(...numeros)); // 3

      const copiaNumeros = [...numeros, 4, 5];
      console.log(copiaNumeros); // [1, 2, 3, 4, 5]
      
        <= Este ejemplo usa el operador spread para pasar los valores del array numeros como argumentos individuales a Math.max, y para crear una copia con nuevos elementos.

Combinar REST y Spread

Puedes usar ambos conceptos juntos para trabajar con argumentos dinámicos y expandir colecciones en JavaScript.

EJEMPLO

      function mezclarArrays(array1, ...otrosElementos) {
        return [...array1, ...otrosElementos];
      }

      const resultado = mezclarArrays([1, 2, 3], 4, 5, 6);
      console.log(resultado); // [1, 2, 3, 4, 5, 6]
      
        <= Este ejemplo combina un array con elementos adicionales utilizando el operador spread y parámetros REST.

Arrow Functions

¿Qué son las Arrow Functions?

Las Arrow Functions son una forma concisa de definir funciones en JavaScript. Se introdujeron en ES6 y ofrecen una sintaxis más corta que las funciones tradicionales, además de no tener su propio contexto this.

SINTAXIS

      // Función tradicional
      function sumar(a, b) {
        return a + b;
      }

      // Arrow Function
      const sumar = (a, b) => a + b;

Ventajas de las Arrow Functions

  • Sintaxis más corta y limpia.
  • No tienen su propio this, por lo que heredan el contexto del entorno donde se definen.

Ejemplos Prácticos

EJEMPLO 1: Función con múltiples argumentos 

      const multiplicar = (a, b) => a * b;
      console.log(multiplicar(5, 3)); // 15
      
        <= Este ejemplo define una Arrow Function que toma dos parámetros y devuelve su producto.

EJEMPLO 2: Función con un solo argumento

      const cuadrado = x => x * x;
      console.log(cuadrado(4)); // 16
      
        <= Cuando hay un solo argumento, los paréntesis son opcionales.

EJEMPLO 3: Función sin argumentos

      const saludar = () => "Hola, mundo!";
      console.log(saludar()); // "Hola, mundo!"
      
        <= Si no hay argumentos, se usan paréntesis vacíos.

EJEMPLO 4: Uso con métodos de arrays

      const numeros = [1, 2, 3, 4, 5];
      const cuadrados = numeros.map(n => n * n);
      console.log(cuadrados); // [1, 4, 9, 16, 25]
      
        <= Este ejemplo utiliza una Arrow Function como callback en el método map.

Cuándo No Usar Arrow Functions

Las Arrow Functions no son adecuadas cuando necesitas una función con su propio contexto this, por ejemplo, en métodos de objetos o constructores.

EJEMPLO

      const objeto = {
        valor: 10,
        metodo: function() {
          return () => this.valor;
        }
      };
      console.log(objeto.metodo()()); // 10
      
        <= Aquí, el contexto this de la Arrow Function se hereda del método del objeto.

Prototipos en JavaScript

Los prototipos son un mecanismo mediante el cual los objetos en JavaScript pueden heredar propiedades y métodos de otros objetos. Cada objeto en JavaScript tiene un prototipo, que actúa como una plantilla de la que puede heredar métodos y propiedades.

¿Cómo funcionan los prototipos?

En JavaScript, un prototipo es un objeto del cual otro objeto hereda. Cuando intentas acceder a una propiedad o método en un objeto, el motor de JavaScript primero busca esa propiedad en el propio objeto. Si no la encuentra, busca en el prototipo del objeto, y así sucesivamente hasta llegar al prototipo base (o Object.prototype).

EJEMPLO

      function Persona(nombre, edad) {
        this.nombre = nombre;
        this.edad = edad;
      }

      // Agregamos un método al prototipo
      Persona.prototype.saludar = function() {
        console.log("Hola, me llamo " + this.nombre);
      };

      // Crear una instancia
      const persona1 = new Persona("Juan", 25);
      persona1.saludar(); // "Hola, me llamo Juan"

      
        <= Aquí, el método saludar no está directamente en persona1, sino en su prototipo (Persona.prototype).

Cadena de Prototipos (Prototype Chain)

La cadena de prototipos es la estructura jerárquica en la que un objeto puede heredar de otro, y este, a su vez, puede heredar de otro más. Si una propiedad o método no se encuentra en el objeto, se buscará en su prototipo, luego en el prototipo del prototipo, y así sucesivamente.

EJEMPLO

      console.log(persona1.toString()); // Llama al método heredado de Object.prototype

      
        <= Aunque no definimos el método toString en Persona, este se encuentra en Object.prototype.

Creación de Objetos con Object.create

También puedes crear objetos directamente con un prototipo específico usando Object.create.

EJEMPLO

      const animal = {
        caminar: function() {
          console.log("El animal está caminando");
        }
      };

      const perro = Object.create(animal);
      perro.ladrar = function() {
        console.log("Guau guau");
      };

      perro.caminar(); // "El animal está caminando"
      perro.ladrar(); // "Guau guau"

      
        <= Aquí, perro hereda el método caminar del objeto animal.

Ventajas de los Prototipos

  • Reutilización de código: Los métodos definidos en el prototipo no se duplican para cada instancia.
  • Mayor eficiencia de memoria: Todas las instancias comparten los mismos métodos y propiedades prototípicas.

Notas importantes

Aunque los prototipos son fundamentales en JavaScript, es importante recordar que la programación moderna tiende a usar la sintaxis de clases (class), que encapsula los prototipos detrás de una sintaxis más amigable.
